Run Security Checks Locally (Before Upload)

Instead of discovering security-check failures only after you upload a plugin version, you can run the exact same checks on your own machine while you develop. A small command-line tool fetches the platform's currently-enabled rules and runs Bandit, detect-secrets and Flake8 locally.

No rule drift: the tool fetches the enabled rules live from the platform (/plugins/api/security-rules/), so what runs locally always matches what the platform enforces. Your plugin source is never uploaded — everything runs on your machine.

Install

The tool lives in the QGIS Plugins Website repository (under tools/security-scan-cli/) and installs directly from GitHub — no PyPI account needed: 


pip install "git+https://github.com/qgis/QGIS-Plugins-Website.git#subdirectory=tools/security-scan-cli"

This installs the qgis-plugin-security command together with the analysis tools (Bandit, detect-secrets, Flake8), pinned to the same versions the platform runs. Pin a specific release by appending @<tag-or-sha> to the repository URL.

Usage
# Scan the current directory
qgis-plugin-security

# Scan a specific plugin directory or an existing .zip
qgis-plugin-security --path path/to/plugin
qgis-plugin-security --path path/to/plugin.zip

# Ignore skippable rules you have reviewed
# (mandatory / critical rules can never be skipped)
qgis-plugin-security --skip B101,E501

# Fail on warnings too, not just critical issues
qgis-plugin-security --strict

# Machine-readable output
qgis-plugin-security --json
Exit Codes
Code Meaning
0 Scan ran, no critical issues — safe to upload.
1 Scan ran, at least one critical issue — blocked, exactly as the platform would block it. Also returned with --strict when warnings are present.
2 Transport/usage error (network, bad --path, invalid response).
Use in CI (GitHub Actions)
- name: QGIS plugin security check
  run: |
    pip install "git+https://github.com/qgis/QGIS-Plugins-Website.git#subdirectory=tools/security-scan-cli"
    qgis-plugin-security --path .

The job fails automatically when critical issues are found (exit code 1).

Use as a Pre-Commit Hook

Add a local hook to your plugin repository's .pre-commit-config.yaml:

repos:
  - repo: local
    hooks:
      - id: qgis-plugin-security
        name: QGIS plugin security check
        entry: qgis-plugin-security
        language: system
        pass_filenames: false
        always_run: true
Config Files Are Respected

Because the CLI runs the same scanner as the platform, any tool config files you bundle in your plugin are honoured locally, exactly as they are on upload:

  • .bandit — skip specific Bandit test IDs or exclude paths.
  • .secrets.baseline — acknowledge known/reviewed secrets so they are not re-reported.
  • .flake8 — ignore specific Flake8 codes or set per-file ignores.

Keep these files at the top level of your plugin (next to metadata.txt). Dotfiles are included automatically, so your config files are picked up without any extra flags. See the Config Files guide for examples.

What Gets Scanned

When you point the CLI at a git working tree, it scans exactly the files git would ship: tracked files plus untracked ones, minus everything matched by your .gitignore (and .git/info/exclude or your global excludes). A virtualenv, build output, or any other ignored folder inside your plugin directory is not zipped or scanned, so local results match what actually ends up in the uploaded plugin, without noise or false positives from third-party code.

Not using git? If the target directory is not a git repository, the CLI falls back to a plain scan that only skips .git, __pycache__ and .mypy_cache. Point --path at a clean copy of your plugin (or an already-built .zip) to avoid scanning a bundled environment.

How Parity With the Platform Is Guaranteed
  • Rules are fetched live from the platform, so there is no local rule list to fall out of date.
  • Scanner code shipped by the tool is the same single source file used by the platform.
  • Tool versions are pinned to match the platform; the endpoint also reports the platform's actual versions, so the tool warns you if your locally-installed Bandit, detect-secrets or Flake8 differ.

Note: local results are a faithful preview, not a guarantee of approval. The authoritative scan still runs when you upload the version.
